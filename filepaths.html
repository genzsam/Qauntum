HTML File Paths

Introduction

In HTML, file paths are used to specify the location of files (such as images, stylesheets, or scripts) that are linked or embedded in a web page. Understanding how to write file paths correctly is essential for building effective websites.

Absolute vs. Relative Paths

There are two main types of file paths in HTML:

  • Absolute Paths: These paths specify the complete address of a file or resource, typically starting from the root directory of the website. For example: /images/pic.jpg
  • Relative Paths: These paths specify the location of a file relative to the current web page's location. Relative paths are more flexible and commonly used. For example: images/pic.jpg

Common File Paths

Here are some common file paths you might encounter:

            <img src="images/pic.jpg" alt="A picture">
            <link rel="stylesheet" href="styles/main.css">
            <script src="scripts/app.js"></script>

Root Directory

The root directory of a website is the highest-level directory, often represented as "/." Absolute paths start from this directory.

Directory Levels

When navigating through directories, you can use forward slashes (/) to separate directory names. For example, images/photos/pic.jpg represents a file located in the "pic.jpg" file within the "photos" directory, which is inside the "images" directory.
